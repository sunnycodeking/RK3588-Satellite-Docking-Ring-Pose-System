# 基于 RK3588 的卫星对接环双目跟踪和位姿测量系统

本工程面向在轨服务近距离巡检与对接环位姿测量任务，整体采用 **RK3588 嵌入式服务端 + Ubuntu Qt 上位机客户端 + 检测跟踪算法 + 双目位姿测量算法** 的分层架构。系统可完成双目视频预览、目标检测跟踪、PWM 舵机控制、传感器状态显示、位姿测量结果回传以及上位机可视化显示等功能。

## 目录结构

```text
.
├── rk3588_tcp_server      # RK3588 服务端控制程序
├── sta_qt_client_v2       # Ubuntu Qt 上位机客户端程序
├── yolov8_ellipse         # 目标检测跟踪程序工程代码
└── yolov8_pose            # 双目位姿测量程序工程代码
```

---

## 1. rk3588_tcp_server

`rk3588_tcp_server` 为运行在 RK3588 板端的服务端控制程序，是整个系统的软件调度中心。

主要功能包括：

- 启动 TCP Server，默认监听 `8888` 端口，等待 Ubuntu Qt 上位机连接；
- 接收并解析上位机发送的 JSON 控制命令，例如 `HELLO`、`START_MODE`、`STOP_MODE`、`SERVO_MANUAL`、`MEASURE_POSE`、`GET_SENSOR` 等；
- 统一管理系统工作模式，包括空闲模式、原始视频预览模式和检测跟踪模式；
- 启动、停止和管理子进程，例如视频预览进程、检测跟踪进程和位姿测量程序；
- 负责模式互斥控制，避免视频预览、检测跟踪、手动舵机控制等功能同时占用摄像头或 PWM 资源；
- 通过 sysfs PWM 节点控制二轴舵机云台，实现左转、右转、俯仰步进和归中控制；
- 读取或生成光照、平台温度等传感器状态信息，并通过 TCP JSON 返回上位机；
- 接收上位机位姿测量命令后，调度检测跟踪程序保存原始双目图像，并调用 `yolov8_pose` 完成位姿解算；
- 将板端日志、检测跟踪状态、舵机 PWM 占空比、传感器数据、位姿文本结果等信息回传至上位机；
- 通过 UDP/RTP/JPEG 链路向上位机发送实时视频流或位姿结果图。

该目录中的程序相当于板端“总控程序”，负责把摄像头、舵机、传感器、检测跟踪算法、位姿测量算法和上位机通信连接起来。

## 2. sta_qt_client_v2

`sta_qt_client_v2` 为运行在 Ubuntu PC 端的 Qt 上位机客户端程序，主要用于远程控制 RK3588 板端并显示系统运行结果。

主要功能包括：

- 提供登录、连接和主控制界面；
- 输入 RK3588 板端 IP 地址和 TCP 端口，建立与板端服务器的 TCP 连接；
- 发送 JSON 控制命令，实现视频预览、检测跟踪、手动舵机控制、传感器刷新和单次位姿测量等操作；
- 接收 RK3588 服务端返回的 `STATUS`、`HEARTBEAT`、`SENSOR`、`DETECT_STATUS`、`SERVO_STATUS`、`POSE_RESULT` 和 `LOG` 等消息；
- 通过 UDP/RTP/JPEG 接收实时视频流，默认视频端口为 `5000`；
- 通过 UDP/RTP/JPEG 接收位姿测量结果图，默认位姿图像端口为 `5002`；
- 在界面中显示实时视频、跟踪状态、舵机 PWM 占空比、光照强度、平台温度、板端日志、位姿文本和位姿结果图；
- 提供左转、右转、向上、向下、归中等手动云台控制按钮；
- 根据系统当前模式控制按钮使能关系，避免手动控制与自动跟踪控制冲突。

该目录中的程序相当于系统的人机交互端，用于比赛演示、功能控制、状态监控和结果可视化。

## 3. yolov8_ellipse

`yolov8_ellipse` 为检测跟踪程序工程代码，运行在 RK3588 板端，主要实现卫星对接环目标检测、KCF 跟踪和云台自动控制。

主要功能包括：

- 读取双目相机输出的图像流，获取对接环目标画面；
- 调用 YOLOv8/RKNN 模型对卫星对接环目标进行检测；
- 从检测结果中筛选最优目标框，计算目标中心与图像中心之间的横向和纵向偏移量；
- 使用 KCF 跟踪器完成目标帧间跟踪，减少持续调用目标检测模型带来的计算开销；
- 实现 DETECTING、TRACKING、PREDICTING 三状态协同控制逻辑：
  - `DETECTING`：目标搜索状态，主要依靠 YOLO 检测目标；
  - `TRACKING`：稳定跟踪状态，主要依靠 KCF 跟踪目标，并周期性使用 YOLO 进行校正；
  - `PREDICTING`：短时预测与重捕获状态，在目标短时丢失或跟踪不稳定时提高检测频率，尝试重新捕获目标；
- 根据目标偏移量计算二轴舵机云台的 PWM 控制量，使目标尽量保持在图像中心附近；
- 将检测框、跟踪状态、目标偏移量、PWM 占空比等信息输出给 RK3588 服务端；
- 响应服务端发送的快照请求，保存未叠加检测框、跟踪框和文字信息的原始双目图像，为后续位姿测量提供输入。

该目录中的程序是系统的“目标发现与自动跟踪模块”，负责完成对接环目标锁定和云台跟随控制。

## 4. yolov8_pose

`yolov8_pose` 为位姿测量工程代码，运行在 RK3588 板端，主要实现基于双目视觉的卫星对接环空间位置和姿态测量。

主要功能包括：

- 读取检测跟踪程序保存的原始双目图像，例如 `/tmp/stereo_pose_input.jpg`；
- 将双目拼接图像拆分为左图和右图；
- 调用目标检测模型定位对接环区域，获得左右图像中的目标 ROI；
- 在 ROI 区域内进行图像预处理、边缘提取、轮廓筛选和椭圆拟合；
- 根据椭圆参数、相机内参和对接环半径先验，分别求解左、右视图下的单目空间圆候选位姿；
- 针对空间圆单目投影存在二义性的问题，利用双目外参、圆心位置一致性和法向量一致性筛选真实解；
- 输出对接环中心三维坐标、圆平面法向量和姿态角等位姿测量结果；
- 生成位姿文本结果 `result.txt` 和结果图 `result.jpg`；
- 由 `rk3588_tcp_server` 读取并回传位姿文本，同时将结果图通过 UDP/RTP/JPEG 发送到 Qt 上位机显示。

该目录中的程序是系统的“空间位姿测量模块”，负责根据原始双目图像完成对接环的三维位置与姿态解算。

## 系统工作流程概述

1. 启动 RK3588 板端 `rk3588_tcp_server` 服务端程序；
2. 打开 Ubuntu 端 `sta_qt_client_v2` 上位机，输入 RK3588 IP 和端口并建立连接；
3. 上位机发送视频预览命令，RK3588 回传原始双目视频流；
4. 上位机发送检测跟踪命令，RK3588 启动 `yolov8_ellipse`，完成目标检测、KCF 跟踪和云台自动控制；
5. 在检测跟踪运行时，上位机发送位姿测量命令；
6. `rk3588_tcp_server` 请求 `yolov8_ellipse` 保存未标注原始双目帧；
7. `rk3588_tcp_server` 调用 `yolov8_pose` 完成双目位姿测量；
8. 位姿文本结果和结果图返回 Qt 上位机显示。

---

## 主要通信端口

| 功能 | 协议 | 默认端口 | 说明 |
|---|---|---:|---|
| 控制命令与状态回传 | TCP JSON | 8888 | Qt 上位机与 RK3588 服务端通信 |
| 实时视频流 | UDP/RTP/JPEG | 5000 | 显示原始视频或检测跟踪视频 |
| 位姿结果图 | UDP/RTP/JPEG | 5002 | 显示 `result.jpg` 位姿测量结果图 |

---

## 注意事项

- 运行前需保证 RK3588 和 Ubuntu 上位机处于同一局域网；
- Qt 上位机连接时需填写 RK3588 当前 IP 地址；
- 若更换网络环境，需要重新确认 RK3588 IP 和上位机 IP；
